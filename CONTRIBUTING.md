# Contributing

Thank you for helping improve On-Prem RAG Assistant.

1. Open an issue before large architectural changes.
2. Create a focused branch and keep unrelated changes separate.
3. Install development dependencies with `python -m pip install -e ".[dev]"`.
4. Run `ruff check src tests` and `pytest` before opening a pull request.
5. Include tests for bug fixes and behavior changes.
6. Do not commit private documents, chat databases, vector stores, logs, or secrets.

Bug reports should include the operating system, Python and Ollama versions,
model names, reproduction steps, expected behavior, and relevant redacted logs.
